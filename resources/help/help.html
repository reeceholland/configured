CONFIGURED Help

Welcome to CONFIGURED, a robotics configuration studio for creating, editing, saving, and exporting structured project configurations.

This guide covers the main workflow, project metadata, Git integration, and export features.

1. Getting Started

Create a new project

  • Select New Project from the home screen.
  • Enter the project metadata when prompted.
  • If the project is Git-managed, choose a parent folder for the project.
  • The project will then be created and opened in the editor.

Open an existing project

  • Select Open Project from the home screen.
  • Choose an existing .configured project file.
  • The project will load into the editor.

2. Editor Overview

The editor is where you manage your project structure and configuration data.

  • Add Child adds a new child item beneath the selected item.
  • Remove Selected removes the currently selected item.
  • The editor allows you to build a hierarchical configuration tree.
  • Changes in the editor can be saved back into the project file.

3. Project Menu

  • Save Project saves the current project to a .configured file.
  • Project Metadata opens the metadata dialog for project details.
  • Home returns to the home screen.
Returning to the home screen may discard unsaved changes. Save first if you want to keep your work.

4. Project Metadata

Project metadata stores identifying information for the current project.

  • Project Name is used in the application title and export metadata.
  • Version is used for non-Git-managed projects.
  • Git Managed indicates whether the project is linked to a Git repository.
  • Git Commit Hash can be stored for Git-managed projects.

5. Saving Projects

Projects are saved as .configured files.

  • Use Save Project to save the current state of the project.
  • Saving before Git actions ensures the latest project data is written to disk.
  • Saving before export ensures the export reflects the latest project state.
It is good practice to save before exporting, checking Git status, or creating a Git commit.

6. Git Integration

CONFIGURED supports basic Git integration for Git-managed projects.

Git Init

  • Initializes a Git repository in the project folder.
  • Marks the project as Git-managed.

Git Status

  • Shows the current working tree status for the project repository.

Git Commit

  • Prompts for a commit message.
  • Saves the project before staging and committing changes.
  • Attempts to store the latest commit hash in project metadata.

Git Identity

  • Opens the Git identity configuration dialog.
  • Use this to configure Git user details when needed.
Git actions are only available for saved projects with a valid project working directory.

7. Exporting

Projects can be exported in both JSON and XML formats.

JSON Export

  • Exports collected project parameters to a JSON file.
  • Includes project metadata such as project name and Git/version information.

XML Export

  • Exports collected project parameters to an XML file.
  • Includes the same project metadata as JSON export.

Exported metadata may include:

  • Project name
  • Git-managed state
  • Git commit hash for Git-managed projects
  • Version for non-Git-managed projects

8. Recommended Workflow

  1. Create or open a project.
  2. Edit the configuration tree.
  3. Update project metadata if needed.
  4. Save the project.
  5. Use Git actions if the project is Git-managed.
  6. Export to JSON or XML when needed.

9. Troubleshooting

Git actions are unavailable

  • Make sure the project has been saved.
  • Make sure Git is installed and available in the system path.
  • Make sure the project folder is a valid Git repository if using status or commit.

Export fails

  • Check that a project is open.
  • Check that the chosen file location is writable.
  • Save the project before exporting.

Project metadata looks out of date

  • Save the project again.
  • For Git-managed projects, ensure the repository contains a valid commit history.

10. About

CONFIGURED is designed to provide a simple desktop interface for managing structured robotics configuration projects and exporting them into standard formats.
